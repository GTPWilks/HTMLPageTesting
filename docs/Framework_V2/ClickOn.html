to RelativeTo SearchType, SearchObject, ScreenArea, SecondsToWait, SecondsToPause, Direction, Distance, Optionalfrom, ResetMouse: false
 
Click a distance from the edge of a rectangle which bounds an element, either image or text.
the rectangle edge selected is based on the direction Exit. Up is Top, Left is Left, Down is Bottom and Right is Right.
By default the center of each edge is used as the start or from point. the from point can
optionally be Exit, either left or right for Horizontal edges; top or bottom for vertical edges.
As we are specIfying a direction, distance should always be a positive number so in all instances
the absolute value of distance will be used. Will LogError when attempting to click on, or within, SearchObject's rectangle.
Due to the absolute value issue above this will only be possible If a distance of zero is used.
 Params SearchType -- Image or Text search, must specIfy which
SearchObject -- the Image, or Text, you want to click relative to
ScreenArea -- valid ScreenPart for finding the TextForSR (required)
SecondsToWait -- the number of seconds for the 'WaitFor' (default is 10)
SecondsToPause -- hard wait before attempting click action
Direction -- "Up", "Down", "Left", "Right" (Not case sensitive)
Distance -- Number of pixels in "Direction"
Optionalfrom -- Optional field.
If Direction is "Up" or "Down" then only "Left" or "Right" can be used.
If Direction is "Left" or "Right" then only "Top" or "Bottom" can be used.
If this field is left empty then "Center" is used
